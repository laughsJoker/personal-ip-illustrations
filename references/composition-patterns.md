# Composition Patterns

Use one primary composition per image. Do not combine too many structures.

## Single-Viewpoint Metaphor

Best for one sentence or one claim.

Pattern:

```text
abstract problem -> physical object -> IP action -> simple result
```

Example:

```text
big requirement -> tangled balloon -> character cuts it -> small task card
```

## Workflow

Best for process, method, automation, AI workflow.

Pattern:

```text
input -> IP processes -> output
```

Keep 3-5 core elements and 1-3 arrows. Avoid formal flowchart styling.

## Before / After

Best for transformation.

Pattern:

```text
left: messy / right: clear
left: forcing / right: decomposed
left: anxious / right: stable
```

The IP can be the bridge or repair worker between states.

## Character State

Best for emotion, workplace, parenting, growth.

Use 2-4 small states. Each state gets one short label.

## Comic Panels

Best for story-like content.

Use 2-4 panels. One action per panel. Avoid long dialogue.

## Knowledge Card

Best for social media.

This mode may use a short title. For article illustrations, avoid titles.

Pattern:

```text
short title + IP action + 3 short points
```

## Cover / Hero Image

Best for article covers, social media headers, course thumbnails.

The IP character should be the focal point — large, centered or slightly off-center, full of personality. This is the only composition where a title is expected.

Pattern:

```text
big IP character (50%-70% of canvas) + short bold title + minimal background elements
```

Rules:
- Title should be 4-12 Chinese characters, placed at top or side.
- Background stays clean — at most 1-2 environment objects in low saturation.
- IP character expression should be "default" or "acting" (energetic, welcoming).
- Do not clutter with multiple props — 1 signature prop max.
- Aspect ratio may be 1:1 (social) or 3:4 (story/cover) instead of 16:9.

## Multi-IP Scene

Use only when explicitly requested.

Give each IP a different position, action, and role. Keep their appearance anchors visible. Avoid more than 2-3 IPs in one image.
